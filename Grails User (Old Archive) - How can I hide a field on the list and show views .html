How can I hide a field on the list and show views?

61 posts
I am using Grails 2.2.2 and adding display: false to a field on a domain class (with dynamic scaffolding) does not hide the field on the list and show views, where I do not want it displayed, but rather on the create and edit views, where I actually do (I want the user to be able to create a domain instance and supply a value for this field, only without it being displayed to someone who’s neither creating nor editing the instance).

Is there another way to get this effect with dynamic scaffolding, or am I just doing something wrong?

Hi,

I have modified the scaffolding templates to have this behaviour.
Original list.gsp (partial): 
					<%  excludedProps = Event.allEvents.toList() << 'id' << 'version'
						allowedNames = domainClass.persistentProperties*.name << 'dateCreated' << 'lastUpdated'
						props = domainClass.properties.findAll { allowedNames.contains(it.name) && !excludedProps.contains(it.name) && it.type != null && !Collection.isAssignableFrom(it.type) }
						Collections.sort(props, comparator.constructors[0].newInstance([domainClass] as Object[]))
						props.eachWithIndex { p, i ->
							if (i < 6) {
								if (p.isAssociation()) { %>
						<th><g:message code="${domainClass.propertyName}.${p.name}.label" default="${p.naturalName}" /></th>
					<%      } else { %>
						<g:sortableColumn property="${p.name}" title="\${message(code: '${domainClass.propertyName}.${p.name}.label', default: '${p.naturalName}')}" />
					<%  }   }   } %>

Modified list.gsp (partial): 
<% excludedProps = Event.allEvents.toList() << 'id' << 'version'
          allowedNames = domainClass.persistentProperties*.name << 'dateCreated' << 'lastUpdated'
          props = domainClass.properties.findAll { allowedNames.contains(it.name) && !excludedProps.contains(it.name) && it.type != null && !Collection.isAssignableFrom(it.type) }
          Collections.sort(props, comparator.constructors[0].newInstance([domainClass] as Object[]))
          props.eachWithIndex { p, i ->
            boolean displayList = true
            if (hasHibernate) {
              cp = domainClass.constrainedProperties[p.name]
              displayList = (cp && cp.attributes ? cp.attributes['displayList'] : true)
            }
            if (displayList) {
              if (p.isAssociation()) { %>
          <th><g:message code="${domainClass.propertyName}.${p.name}.label" default="${p.naturalName}"/></th>
          <% } else { %>
          <g:sortableColumn property="${p.name}"
                            title="\${message(code: '${domainClass.propertyName}.${p.name}.label', default: '${p.naturalName}')}"
                            params="\${filterParams}"/>
          <% }
          }
          } %>

so in my case I extended to support the 'displayList' attribute
displayList = (cp && cp.attributes ? cp.attributes['displayList'] : true)

and then in my domain class, I end up with:
 static constraints = {
        name(nullable: false, blank: false, unique: true)
        invisibleName(display: false, attributes: [displayList: false])
    }
